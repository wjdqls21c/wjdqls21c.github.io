# 방명록 켜기 (Firebase, 무료)

방명록은 **Firebase**에 글을 저장합니다. GitHub Pages만으로는 방문자가 글을 남길 수 없어서, 이 단계가 필요해요.

---

## 1) Firebase 프로젝트 만들기

1. **https://console.firebase.google.com** 접속
2. **프로젝트 추가** → 이름 예: `bbinking-guestbook`
3. Google Analytics는 **끄거나 켜도 됨** (선택)

---

## 2) 웹 앱 등록

1. 프로젝트 안에서 **웹** 아이콘 `</>` 클릭
2. 앱 닉네임: `내 홈페이지`
3. **앱 등록** → **Firebase SDK** 설정 값이 나옴
4. `firebaseConfig` 객체 내용을 복사

---

## 3) 사이트에 붙이기

`js/firebase-config.js` 파일을 열고 복사한 값을 넣습니다:

```javascript
export const firebaseConfig = {
  apiKey: "AIza...",
  authDomain: "프로젝트ID.firebaseapp.com",
  projectId: "프로젝트ID",
  storageBucket: "프로젝트ID.firebasestorage.app",
  messagingSenderId: "123...",
  appId: "1:123...:web:...",
};
```

저장 후 GitHub에 올립니다:

```powershell
cd "$env:USERPROFILE\OneDrive\바탕 화면\커서 공부용"
git add js/firebase-config.js
git commit -m "Configure Firebase for guestbook"
git push
```

---

## 4) Firestore 데이터베이스 만들기

1. Firebase 콘솔 → **빌드** → **Firestore Database**
2. **데이터베이스 만들기** → **프로덕션 모드**로 시작 (규칙은 다음에 수정)
3. 위치: **asia-northeast3 (서울)** 추천

---

## 5) 보안 규칙 (중요)

Firestore → **규칙** 탭 → 아래로 **교체** → **게시**:

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /guestbook/{messageId} {
      allow read: if true;
      allow create: if request.resource.data.keys().hasAll(['name', 'message', 'createdAt'])
        && request.resource.data.name is string
        && request.resource.data.message is string
        && request.resource.data.name.size() > 0
        && request.resource.data.name.size() <= 20
        && request.resource.data.message.size() > 0
        && request.resource.data.message.size() <= 300;
      allow update, delete: if false;
    }
  }
}
```

- 누구나 **읽기·새 글 쓰기** 가능
- **수정·삭제**는 불가 (스팸 방지 기본)

## 문제 해결

| 증상 | 해결 |
|------|------|
| 설정 안내가 계속 보임 | `firebase-config.js` 값이 비어 있지 않은지 확인 |
| 등록 실패 | Firestore 규칙을 위 내용으로 게시했는지 확인 |
| 목록이 안 보임 | Firestore에 `guestbook` 컬렉션이 생겼는지 확인 |

설정이 어려우면 Cursor에 **「방명록 Firebase 설정 도와줘」** 라고 하면 됩니다.
