# 관리자 페이지 설정 (방명록 삭제)

관리 주소: **https://wjdqls21c.github.io/admin.html**

---

## 1) 관리자 Gmail 넣기

`js/firebase-config.js` 파일을 열고:

```javascript
export const adminEmail = "본인@gmail.com";
```

Google 로그인에 쓰는 **Gmail과 똑같이** 적으세요.

저장 후 GitHub에 올리기:

```powershell
cd "$env:USERPROFILE\OneDrive\바탕 화면\커서 공부용"
git add js/firebase-config.js
git commit -m "Set admin email"
git push
```

---

## 2) Firebase에서 Google 로그인 켜기

1. **https://console.firebase.google.com** → 프로젝트 `bbakbinking`
2. **빌드** → **Authentication** → **시작하기** (처음이면)
3. **로그인 방법** → **Google** → **사용 설정** → 저장
4. **설정** 탭 → **승인된 도메인**에 아래가 있는지 확인:
   - `wjdqls21c.github.io`
   - 없으면 **도메인 추가**

---

## 3) Firestore 규칙 (삭제 권한)

1. **Firestore Database** → **규칙**
2. `firestore.rules` 파일 내용을 참고해, **`YOUR_GMAIL@gmail.com`** 을 본인 Gmail로 **두 군데 모두** 맞춤:
   - `firebase-config.js` 의 `adminEmail`
   - 규칙의 `request.auth.token.email == '...'`
3. **게시**

예시 (이메일만 본인 것으로 바꿈):

```
allow delete: if request.auth != null
  && request.auth.token.email == 'wjdql@gmail.com';
```

---

## 4) 사용 방법

1. **https://wjdqls21c.github.io/admin.html** 접속
2. **Google로 로그인**
3. 방명 목록에서 **삭제** 클릭 → 확인

다른 Google 계정으로 로그인하면 **권한이 없습니다** 가 나옵니다.

---

## 문제 해결

| 증상 | 해결 |
|------|------|
| 로그인 팝업 안 됨 | 브라우저 팝업 차단 해제 |
| 권한 없음 | `adminEmail` 과 로그인 Gmail이 같은지 |
| 삭제 실패 | Firestore 규칙에 `delete` 와 이메일이 맞는지, **게시** 했는지 |
| 설정 필요 화면 | `adminEmail` 이 비어 있지 않은지 |
